# 废品/布碎采购管理系统 - 数据库设计

## 概述
本系统采用纯前端架构，使用 localStorage 进行数据持久化存储。以下是详细的数据库（存储）设计方案。

---

## 数据模型设计

### 1. 用户表 (users)
```javascript
{
  id: 'user-001',              // 用户ID，唯一标识
  username: '皖盛布碎',         // 用户名（登录账号）
  password: '123456',          // 密码（实际应加密存储）
  name: '管理员',              // 用户姓名
  phone: '13800138000',        // 手机号
  email: 'admin@wsbs.com',      // 邮箱
  avatar: '',                   // 头像URL
  department: '财务部',          // 部门
  role: 'admin',                // 角色：admin, staff, guest
  permissions: ['all'],         // 权限列表
  status: 'active',             // 状态：active, disabled
  createdAt: '2024-01-01T00:00:00Z',  // 创建时间
  updatedAt: '2024-01-01T00:00:00Z'   // 更新时间
}
```

**索引建议：**
- username: 唯一索引
- status: 普通索引

      id: 'item-001',

## 单据编号规则

### 进货单编号
格式：`CG` + `YYYYMMDD` + `4位随机数`
示例：`CG202404111234`

### 出货单编号
格式：`XS` + `YYYYMMDD` + `4位随机数`
示例：`XS202404111234`

## 存储键名约定

所有存储数据使用 `wsbs_` 前缀：

```javascript
// 用户认证
wsbs_token              // 登录token
wsbs_user               // 用户信息

// 业务数据
wsbs_customers          // 客户列表
wsbs_categories         // 品类列表
wsbs_fabrics            // 布料列表
wsbs_bills              // 单据列表

// 草稿与状态
wsbs_draft_bill         // 当前编辑的草稿
wsbs_recent_customers   // 最近使用客户
wsbs_recent_fabrics     // 最近使用布料
wsbs_app_config         // 应用配置

// 统计缓存
wsbs_stats_cache        // 统计数据缓存
```

## 性能优化建议

### 1. 分页加载
单据列表使用分页加载，每页 20-50 条记录

### 2. 索引利用
对常用查询字段建立索引（在内存中优化）

### 3. 缓存策略
- 客户、品类、布料数据全量缓存
- 统计数据缓存 5 分钟
- 单据详情缓存 1 分钟

### 4. 数据归档
- 超过 1 年的单据可归档
- 归档数据仍可查询，但不参与默认统计
