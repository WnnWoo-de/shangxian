# 废品/布碎采购管理系统 - Web 前端开发文档

## 0. 快速启动（Web 移动端版）

### 0.1 环境要求
本地开发用到 
- Node.js：`>= 18`
- npm：`>= 9`
- 推荐浏览器：Chrome 最新版

### 0.2 启动命令

在项目根目录执行：

- `npm install`：安装依赖
- `npm run dev`：本地开发
- `npm run build`：生产构建
- `npm run preview`：预览生产包
- `npm run lint`：代码检查

## 2. 技术栈与开发目标

### 2.1 技术栈

- 框架：`Vue 3` + `JavaScript`
- 构建工具：`Vite`
- 路由：`Vue Router`
- 状态管理：`Pinia`
- UI 组件库：`Element Plus`
- 网络请求：`Axios` 二次封装
- 图表（统计页可选）：`ECharts`
- 样式方案：`SCSS` + 模块化样式

### 2.2 前端目标

1. 支持高频录单与历史查询，降低操作成本。
2. 保证重量、金额、汇总计算准确。
3. 适配主流桌面浏览器（Chrome/Edge）。
4. 支持草稿恢复、历史筛选、打印/导出。
5. 满足生产部署与可维护性要求。
6. 移动端适配

## 4. 页面规划

### 4.1 页面清单

#### 1）登录页
- 用途：账号密码登录，获取登录态。
- 核心元素：品牌信息、登录表单、记住登录、隐私协议。

#### 2）工作台（首页）
- 用途：业务导航与数据概览。
- 推荐模块：
  - 今日开单数
  - 今日采购金额
  - 快捷入口（开单、客户、品类、历史、统计）

#### 3）客户管理页
- 功能：客户列表、搜索、新增、编辑、停用。
- 交互：表格 + 分页 + 侧边抽屉编辑。

#### 4）品类管理页
- 功能：品类列表、默认价格维护、启停管理。
- 交互：支持行内编辑默认价格。

#### 5）开单页
- 功能：核心录单页面。
- 内容：
  - 单据日期
  - 客户选择
  - 动态明细
  - 汇总栏
  - 保存草稿 / 提交单据

#### 6）单据历史页
- 功能：历史单据筛选与查看。
- 内容：日期范围、客户筛选、品类筛选、汇总与列表。

#### 7）单据详情页
- 功能：展示单据主信息、明细、汇总。
- 操作：打印、导出、分享（按业务需要）。

#### 8）统计页
- 功能：按时间、客户、品类维度进行汇总分析。
- 第一版建议：先做列表统计，再补图表。

#### 9）系统设置页
- 功能：账号信息、修改密码、缓存清理、版本信息。

## 5. 交互设计原则（Web移动端）

### 5.1 效率优先

1. 开单入口固定在导航一级菜单。
2. 录单操作尽量在单页完成，减少弹窗层级。
3. 客户、品类支持关键字搜索与最近使用。
4. 明细新增、删除、价格修改反馈即时。

### 5.2 桌面端体验原则

1. 使用表格与快捷键提升录入效率。
2. 表单区域与明细区域布局清晰。
3. 长列表统一分页，避免一次渲染过多。
4. 危险操作（删除/作废）必须二次确认。

### 5.3 错误提示原则

1. 优先给出可执行提示，如“请先选择客户”。
2. 不向业务用户暴露后端堆栈信息。
3. 网络异常、鉴权失效、保存失败需明确提示。

## 6. 开单页核心实现方案

### 6.1 数据结构建议

```ts
interface BillForm {
  id?: string
  billDate: string
  customerId: string
  customerName: string
  note: string
  items: BillItemForm[]
}

interface BillItemForm {
  categoryId: string
  categoryName: string
  weightInputText: string
  totalWeight: number
  unitPrice: number
  amount: number
  note: string
}
```

### 6.2 连续称重输入规则

支持以下输入：
- `105 95 92`
- `105,95,92`
- `105+95+92`
- `105，95，92`

处理规则：
1. 将中文逗号、英文逗号、加号、多个空格统一替换为分隔符。
2. 拆分后过滤空值。
3. 每项转数字并校验。
4. 计算总重量并回填。

### 6.3 计算规则

1. 行金额 = `总重量 × 单价`
2. 总重量 = 所有有效行 `totalWeight` 求和
3. 总金额 = 所有有效行 `amount` 求和
4. 金额统一保留两位小数（建议内部用“分”计算）

### 6.4 价格自动带出

优先级：
1. 客户+品类最近成交价
2. 品类默认单价
3. 空值（手动输入）

### 6.5 草稿机制

触发时机：
1. 点击“保存草稿”
2. 路由离开时
3. 页面关闭前（`beforeunload`）

恢复逻辑：
1. 进入开单页检测本地草稿
2. 弹窗询问是否恢复
3. 提交成功后清空草稿
